# 软件应用与开发类技术设计文档
# 第一章 需求分析
## 1.1 开发原因
射击训练物联网设备长期面临"有数据、无智能"的核心矛盾：

1. **人工判环效率低**：肉眼判读3-5分钟/张，误差率5-10%
2. **动作问题难定位**：轨迹与压力数据已采集，但缺乏AI诊断手段
3. **训练数据无沉淀**：历史数据分散在本地，无法长期追踪与横向对比
4. **军警合规门槛高**：进口系统数据存海外，不符合《数据安全法》；国产专用系统10万+，部署复杂

## 1.2 目标用户
+ **核心用户**：射击俱乐部（约1,200家）、体校射击队（约500所）、军警训练单位
+ **次要用户**：专业运动员、业余爱好者

## 1.3 竞品对标分析
| 对比维度 | 传统人工 | 进口高端(Scatt) | 国内军警专用系统 | 本项目 |
| :--- | :--- | :--- | :--- | :--- |
| **核心功能** | 肉眼判读 | 轨迹+压力分析 | 标准化考核+档案 | 靶面+轨迹+压力+AI诊断 |
| **AI智能诊断** | 无 | 无 | 无 | 有（大模型+自研CV） |
| **数据合规** | - | 数据存海外 | 全国产化，封闭系统 | 全国产化存储，支持本地部署 |
| **使用门槛** | 依赖经验 | 需专业培训+专用硬件 | 部署复杂 | 基于现有IoT设备，即装即用 |


**差异化定位**：唯一同时满足"AI智能诊断+国产化合规+基于现有IoT设备即装即用"三个条件的方案。

# 第三章 详细设计
## 3.1 系统整体架构设计
### 3.1.1 端-云协同轻量化架构
采用"端侧重实时、云侧重智能"分工原则，四层架构：

+ **端侧（学生自研）**：蓝牙通信、图像预处理、本地缓存、可视化渲染、弱网自适应
+ **安全层（学生自研）**：AES-128蓝牙加密、TLS 1.3传输加密、API鉴权限流
+ **云侧（学生自研）**：API网关、靶面识别引擎、动作分析引擎、数据持久化、Serverless计算
+ **硬件层（教师自研已授权）**：光电捕捉系统、压力传感系统

| 模块 | 负责方 | 工作量占比 |
| :--- | :---: | :---: |
| 硬件终端 | 教师团队 | 25% |
| 蓝牙通信/端侧预处理/靶面识别/动作分析/云端架构 | 学生团队 | 75% |


**设计原则**：轻量化（Serverless降本90%）、实时性、可靠性（弱网可独立运行）、可扩展。

### 3.1.2 数据流向与关键时序
+ **上行**：传感器 → 蓝牙 → APP → 预处理 → HTTP上传 → AI分析 → 存储
+ **下行**：AI结果 → HTTP响应 → APP → 界面渲染 → 用户展示

| 环节 | 耗时 | 优化手段 |
| :--- | :---: | :--- |
| 蓝牙传输 | <30ms | BLE 5.0高吞吐量，数据包合并 |
| 图像预处理 | <150ms | 多线程并行，Sharp GPU加速 |
| HTTP上传 | <100ms | 压缩后<100KB |
| AI分析 | <700ms | Serverless预留实例预热 |
| 界面渲染 | <50ms | 本地Canvas绘制 |
| **总计** | **≤1秒** | 实测平均0.8秒 |

## 3.4 核心算法与关键技术
### 3.4.1 靶面图像预处理算法
针对射击靶面特点设计自适应质量评估模型：全局缩放至720px/JPEG质量60%，靶心10环区域采用**局部保真策略**。压缩率97%，AI识别准确率保持99%+。

### 3.4.2 多源数据时序同步算法
解决轨迹100Hz与压力1000Hz的时钟漂移问题：连接时NTP校时 → 实时漂移补偿 → 插值对齐。每次击发前重新校时，击发瞬间高频插值（10kHz等效）。同步精度±5ms，优于竞品±10-20ms。

### 3.4.3 环数计算与轨迹分析
**环数计算**：基于着弹点到靶心欧几里得距离，每环间距7.5mm，环数=10.5-(距离/7.5)。

| 环数 | 半径(mm) |  | 环数 | 半径(mm) |
| :---: | :---: | :---: | :---: | :---: |
| 10.9 | ≤3.75 |  | 5.5 | ≤45.0 |
| 10.5 | ≤7.5 |  | 4.5 | ≤52.5 |
| 9.5 | ≤15.0 |  | 3.5 | ≤60.0 |
| 8.5 | ≤22.5 |  | 2.5 | ≤67.5 |
| 7.5 | ≤30.0 |  | 1.5 | ≤75.0 |
| 6.5 | ≤37.5 |  | 0.5 | >75.0 |


**轨迹分段**：击发前2秒（瞄准稳定性）→ 击发前0.5秒（击发准备）→ 击发后1秒（后坐力控制），三色可视化（蓝/红/绿），击发点识别精度±20ms。

### 3.4.4 压力曲线分析算法
**特征提取**：击发点识别（压力下降沿触发）、平稳度（前200ms方差）、果断性评估。

| 错误类型 | 量化阈值 | 改进建议 |
| :--- | :--- | :--- |
| 猛扣扳机 | 前200ms方差>5N²，下降<50ms | 加强预压，匀速扣压 |
| 犹豫击发 | 前500ms波动≥3次（峰谷差>1N） | 果断击发，避免二次调整 |
| 预压不足 | 初始压力<2N（标准3-5N） | 增加预压量 |
| 击发拖沓 | 下降时间>200ms（标准<100ms） | 缩短击发时间 |
