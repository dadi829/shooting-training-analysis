# 🚀 快速部署指南

本指南将帮助您在 10 分钟内部署好射击训练分析系统！

## 📋 准备工作

1. **GitHub 账号** - 用于托管代码和前端
2. **Vercel 账号** - 用于托管后端（可使用 GitHub 登录）
3. **代码已准备好** - 我们已经为您配置好所有部署文件！

---

## 第一步：部署后端到 Vercel ⚡

### 1.1 推送到 GitHub

首先，将代码推送到您的 GitHub 仓库：

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/您的用户名/您的仓库名.git
git push -u origin main
```

### 1.2 部署到 Vercel

1. 访问 [vercel.com](https://vercel.com) 并登录
2. 点击 **"Add New Project"**
3. 选择您刚推送的 GitHub 仓库
4. 保持默认配置，直接点击 **"Deploy"**
5. 等待 1-2 分钟，部署完成！

### 1.3 获取后端地址

部署完成后，您会看到类似这样的地址：
```
https://your-project-name.vercel.app
```

**保存这个地址！** 下一步需要用到。

---

## 第二步：部署前端到 GitHub Pages 🌐

### 2.1 配置 GitHub Secrets

1. 进入您的 GitHub 仓库
2. 点击 **Settings** → **Secrets and Variables** → **Actions**
3. 点击 **New repository secret**
4. 名称填：`BACKEND_URL`
5. 值填：刚才 Vercel 分配的地址（例如 `https://your-project-name.vercel.app`）
6. 点击 **Add secret**

### 2.2 启用 GitHub Pages

1. 在仓库设置中，点击 **Pages**（左侧菜单）
2. 在 **Build and deployment** 部分：
   - **Source**: 选择 `GitHub Actions`
3. 保存设置

### 2.3 触发部署

推送任意代码到 `main` 分支，GitHub Actions 会自动开始部署：

```bash
git commit --allow-empty -m "Trigger deployment"
git push
```

### 2.4 访问您的网站！

部署完成后，在 Pages 设置页面可以看到访问地址，类似：
```
https://您的用户名.github.io/您的仓库名/
```

## 🎯 可选：启用真实 AI 分析

如果您有字节跳动豆包 API Key，可以启用真实 AI 分析：

1. 进入 Vercel 项目的 **Settings** → **Environment Variables**
2. 添加两个环境变量：
   - `DOUBAO_API_KEY`: 您的 API Key
   - `DOUBAO_ENDPOINT`: 您的模型端点（例如 `ep-2024xxxxxx`）
3. 重新部署项目

## 📱 本地开发

如果想在本地运行：

```bash

npm install

# 启动后端（新终端）
cd api
npm install
node index.js

# 启动前端
npm run dev
```

访问 http://localhost:5173 即可使用！

## 🆘 常见问题

**Q: 部署后前端无法连接后端？**
A: 检查 GitHub Secrets 中的 BACKEND_URL 是否正确，不要加末尾的 `/`

**Q: Vercel 部署失败？**
A: 查看 Vercel 部署日志，检查 package.json 中的依赖是否正确

**Q: GitHub Pages 显示 404？**
A: 等待 2-3 分钟，首次部署需要时间生效
