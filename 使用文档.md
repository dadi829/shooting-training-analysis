## 项目概述

气步枪打靶数据分析系统是一个基于AI的智能射击数据分析工具，通过上传射击训练系统截图，系统可以自动分析射击数据，生成详细的分析报告，帮助运动员提升射击水平。

### 主要功能

- **智能图片分析** - 支持多种AI模型分析射击训练截图
- **多模型支持** - 模拟数据、OpenAI、通义千问、豆包
- **数据可视化** - 统计图表、趋势分析、数据表格
- **历史记录** - 完整的历史数据管理、检索、对比
- **数据导出** - CSV格式导出历史数据

## 快速开始

### 1. 安装依赖

#### 后端依赖

```bash
cd backend
npm install
```

#### 前端依赖

```bash
cd ..  # 返回项目根目录
npm install
```

### 2. 配置环境变量

在 `backend` 目录下创建或编辑 `.env` 文件：

```env
# OpenAI API (可选)
OPENAI_API_KEY=your-openai-api-key

# 通义千问 API (可选)
TONGYI_API_KEY=your-tongyi-api-key

# 豆包 API (可选)
DOUBAO_API_KEY=your-doubao-api-key
```

**注意**：
- 如果不配置API密钥，可以使用"模拟数据"模式进行测试
- API密钥只存储在后端，不会暴露在前端

### 3. 启动项目

#### 方式一：使用启动脚本（Windows）

双击项目根目录下的 `start.bat` 文件

#### 方式二：手动启动

**启动后端服务**（终端1）：

```bash
cd backend
npm start
```

**启动前端服务**（终端2）：

```bash
npm run dev
```

### 4. 访问应用

- **前端应用**: http://localhost:5173
- **后端API**: http://localhost:3002
- **健康检查**: http://localhost:3002/api/health

## 详细使用指南

### 智能分析功能

#### 1. 选择AI模型

1. 进入"智能分析"页面
2. 在左侧设置面板中选择AI模型
3. 点击"刷新模型列表"按钮（如需要）

**可用模型说明**：

| 模型 | 说明 | 响应速度 | 推荐场景 |
|------|------|----------|----------|
| 模拟数据 | 内置模拟数据，无需API | 最快（2秒） | 测试和演示 |
| OpenAI | 使用GPT-4V进行分析 | 中等 | 高质量分析 |
| 通义千问 | 阿里云视觉大模型 | 较快 | 国内用户 |
| 豆包 | 字节跳动视觉大模型 | 较快 | 国内用户 |

#### 2. 上传图片

1. 点击"选择图片"按钮或拖拽图片到上传区域
2. 支持格式：JPG、JPEG、PNG
3. 文件大小限制：最大10MB
4. 上传后会显示图片预览

**图片要求**：
- 必须是10米气步枪射击训练系统的完整截图
- 包含左侧靶图和右侧数据表格
- 图片清晰，文字可识别

#### 3. 开始分析

1. 确认图片上传成功
2. 点击"开始AI分析"按钮
3. 观察进度条变化（0% - 100%）
4. 等待分析完成（通常5-50秒）

#### 4. 查看分析结果

分析完成后，系统会显示：

- **轨迹特征分析**：预瞄稳定性、击发瞬间、击发后表现
- **报靶数据统计**：总发数、总分、平均环数、主要偏移方向等
- **常见问题原因**：AI诊断的3个主要问题
- **改进建议**：针对性的3条改进建议

#### 5. 保存记录

1. 分析完成后，点击"保存记录"按钮
2. 在弹出的对话框中填写：
   - 训练类型（可选）
   - 环境信息（可选）
   - 标签（可选，支持自定义）
   - 备注信息（可选）
3. 点击"保存"按钮
4. 保存成功后会自动跳转到历史记录页面

### 历史记录功能

#### 1. 查看历史记录

1. 切换到"历史记录"标签页
2. 系统会显示所有保存的分析记录
3. 记录按时间倒序排列（最新的在最前面）

#### 2. 筛选和搜索

系统提供多种筛选方式：

- **时间范围筛选**：使用日期范围选择器
- **训练类型筛选**：按训练类型过滤
- **分数范围筛选**：按最低/最高总分过滤
- **标签筛选**：选择一个或多个标签
- **关键词搜索**：在备注中搜索关键字

#### 3. 查看记录详情

1. 在记录列表中找到目标记录
2. 点击"查看"按钮
3. 在弹出的详情对话框中查看完整分析结果

#### 4. 对比分析

1. 在记录列表中勾选多个记录（至少2个）
2. 点击"对比分析"按钮
3. 查看对比分析图表和报告

#### 5. 导出数据

1. 选择要导出的记录（可多选）
2. 点击"导出数据"或"导出选中"按钮
3. 选择保存位置
4. 数据会以CSV格式保存

#### 6. 删除记录

1. 找到要删除的记录
2. 点击"删除"按钮
3. 确认删除操作
4. 记录将永久删除（不可恢复）

## AI限制词和使用规范

### 🔒 严格禁止的内容

系统内置严格的内容安全机制，**禁止**处理以下内容：

| 类别 | 禁止事项 | 违规后果 |
|------|----------|----------|
| 图片内容 | 非射击训练系统截图 | AI会拒绝分析并提示错误 |
| 图片内容 | 包含暴力、血腥、敏感内容 | AI会拒绝分析并提示错误 |
| 图片内容 | 包含政治敏感或违法内容 | AI会拒绝分析并提示错误 |
| 图片内容 | 包含个人隐私信息的图片 | AI会拒绝分析并提示错误 |
| 用户输入 | 在备注、标签中输入违法违规内容 | 无法保存记录 |
| 用户输入 | 恶意测试或滥用系统 | 系统可能限制使用 |

### ✅ 正确使用规范

#### 图片上传规范

1. **只上传射击训练系统截图**
   - 必须包含完整的靶图和数据表格
   - 确保文字清晰可识别
   - 不要上传无关的图片

2. **图片质量要求**
   - 分辨率建议：1920x1080 或更高
   - 确保表格文字清晰
   - 避免模糊、反光、过暗或过亮

3. **数据安全注意**
   - 避免上传包含个人敏感信息的截图
   - 如必须上传，建议先进行脱敏处理

#### AI分析使用规范

1. **理解AI的局限性**
   - AI分析仅供参考，不能替代专业教练指导
   - 对于复杂的技术问题，请咨询专业人员
   - 每次分析结果可能有轻微差异，这是正常的

2. **合理使用建议**
   - 建议每次训练后及时分析数据
   - 定期查看历史记录，关注长期趋势
   - 结合AI建议和实际训练情况进行调整
   - 不要过度依赖单次分析结果

3. **性能优化建议**
   - 如果网络较慢，建议使用"模拟数据"模式测试
   - 保存记录时简洁填写备注信息
   - 定期清理不再需要的历史记录，提升响应速度

#### 数据管理规范

1. **历史记录管理**
   - 建议每次训练后及时保存记录
   - 为记录添加合适的标签，方便后续检索
   - 定期导出重要数据进行备份
   - 及时删除无用记录，节省存储空间

2. **数据隐私保护**
   - 历史记录存储在浏览器本地（localStorage）
   - 清理浏览器数据会丢失历史记录
   - 建议定期导出数据进行备份
   - 不要在公共电脑上使用该系统保存敏感数据

### ⚠️ 注意事项

#### 关于响应时间

- **模拟数据模式**：固定2秒响应（包含模拟延迟）
- **真实AI模式**：通常需要10-45秒
- **超时处理**：超过50秒会自动超时，建议重试或使用模拟数据
- **进度显示**：进度条会平滑更新，提供更好的用户体验

#### 关于分析结果

- AI分析结果基于视觉识别，可能存在误差
- 建议结合实际训练情况进行判断
- 如发现分析结果明显错误，可以重新上传分析
- 多次分析取平均可以提高准确性

#### 关于系统稳定性

- 如果前端显示"无法连接后端"，请检查后端服务是否启动
- 如果API调用失败，建议先使用模拟数据模式
- 遇到问题可以尝试刷新页面或重启服务
- 如持续出现问题，请查看控制台日志并报告错误

## 常见问题排查

### 后端服务相关

**Q: 后端服务无法启动怎么办？**

A: 请检查以下几点：
1. 确认端口3002没有被其他程序占用
2. 确认已在backend目录下运行`npm install`
3. 确认Node.js版本≥16.0
4. 查看终端中的错误信息，根据提示解决

**Q: 后端服务启动了，但前端无法连接？**

A: 请检查：
1. 确认后端地址是 http://localhost:3002
2. 检查浏览器控制台是否有CORS错误
3. 尝试访问 http://localhost:3002/api/health 测试后端是否正常

### AI分析相关

**Q: AI分析一直显示超时？**

A: 建议：
1. 检查网络连接是否稳定
2. 尝试使用"模拟数据"模式测试系统是否正常
3. 如网络较慢，可以考虑多次重试
4. 确认API密钥配置正确且有效

**Q: AI分析结果不准确怎么办？**

A: 建议：
1. 确保上传的图片清晰、完整
2. 尝试重新上传同一张图片再次分析
3. 可以尝试使用不同的AI模型进行对比
4. 记住AI分析仅供参考，结合实际情况判断

**Q: 保存记录后，历史记录页面没有显示？**

A: 请检查：
1. 确认保存操作是否成功（会有成功提示）
2. 检查是否应用了筛选条件，导致记录被过滤
3. 尝试刷新页面重新加载
4. 清除所有筛选条件，查看是否显示

### 历史记录相关

**Q: 历史记录会丢失吗？**

A: 请注意：
1. 历史记录存储在浏览器localStorage中
2. 清理浏览器缓存或数据会导致记录丢失
3. 建议定期导出重要数据进行备份
4. 如果更换浏览器或设备，需要重新导入数据

**Q: 如何备份和恢复历史数据？**

A: 目前支持：
1. 使用导出功能将数据保存为CSV文件（备份）
2. 后续版本会增加数据导入功能
3. 重要数据建议同时截图或复制保存

### 前端使用相关

**Q: 页面显示不正常或按钮没有反应？**

A: 建议：
1. 刷新页面重新加载
2. 清除浏览器缓存
3. 使用最新版本的Chrome或Firefox浏览器
4. 检查浏览器控制台是否有错误信息

**Q: 上传的图片不显示怎么办？**

A: 请检查：
1. 图片格式是否为JPG/JPEG/PNG
2. 文件大小是否超过10MB
3. 尝试使用其他图片测试
