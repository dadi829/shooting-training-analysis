## 一、作品简介（100字以内）

本系统是一款基于Web的射击训练分析管理平台，采用React+Node.js前后端分离架构，集成字节跳动豆包大模型API实现靶面图像智能识别、环数自动计算、射击姿势分析等功能。支持教练/学员双角色协作，提供训练数据可视化展示，通过AI量化分析实现射击训练从经验驱动向数据驱动的科学化转变。

## 二、AI工具使用说明

### 作品所使用的AI工具或平台
字节跳动豆包大模型API

### AI在作品中的应用说明（没有则忽略）
本作品使用字节跳动豆包大模型API进行靶面图像智能分析。用户上传射击靶面图片后，系统调用豆包多模态大模型API，对图片进行识别分析，自动提取弹着点坐标、计算射击环数、评估射击姿势稳定性，并生成个性化训练改进建议。AI分析结果以结构化JSON格式返回，经系统解析后以可视化方式呈现给用户。AI功能占作品核心功能的约30%，主要应用于图像识别与智能分析模块。

## 四、作品安装说明

### 环境要求

- **操作系统**: Windows 10/11、macOS、Linux
- **Node.js**: 版本 16.0 或更高
- **内存**: 4GB 及以上
- **浏览器**: Chrome 90+ / Edge 90+ / Firefox 88+
- **网络**: 稳定的互联网连接（用于AI分析功能）

### 安装步骤

#### 1. 获取项目代码

```bash
git clone <repository-url>
cd web-ai
```

#### 2. 后端服务部署

```bash
cd backend
npm install
```

创建 `.env` 文件并配置API密钥：
```
DOUBAO_API_KEY=your_api_key_here
PORT=3002
```

启动后端服务：
```bash
node server.js
```

后端服务默认运行在 http://localhost:3002

#### 3. 前端应用部署

```bash
cd ..
npm install
npm run dev
```

前端开发服务器默认运行在 http://localhost:5173

#### 4. 访问应用

打开浏览器，访问 http://localhost:5173 即可使用系统。

### 默认账号

系统首次使用时，可注册教练账号或学员账号。学员注册时需要选择关联的教练。

## 五、设计思路

### 1. 整体架构设计

系统采用经典的三层架构设计：

```
前端层 (React SPA)
    ↕ RESTful API
后端层 (Node.js + Express)
    ↕ HTTP/HTTPS
AI服务层 (大模型API)
```

#### 1.1 前端架构

- **框架选择**: React 18 + TypeScript 5
  - 组件化开发，代码可维护性强
  - TypeScript提供类型安全，减少运行时错误
  - 丰富的生态系统和社区支持

- **构建工具**: Vite 5
  - 快速的开发服务器启动（秒级）
  - 即时的模块热更新（HMR）
  - 优化的生产构建输出

- **UI组件库**: Ant Design 5
  - 企业级UI设计规范
  - 丰富的组件覆盖常见业务场景
  - 完善的TypeScript类型定义

#### 1.2 后端架构

- **Web框架**: Express 4
  - 轻量灵活，中间件生态丰富
  - 路由清晰，易于维护扩展
  - 性能优异，适合I/O密集型应用

- **文件处理**: Multer + Sharp
  - Multer处理多文件上传
  - Sharp进行图像压缩和预处理
  - 优化AI分析前的图像质量

- **数据存储**: JSON文件存储
  - 初期采用轻量级方案
  - 便于部署和维护
  - 可平滑迁移至数据库

#### 1.3 AI集成架构

- **大模型API**: 字节跳动豆包
  - 多模态能力，支持图像分析
  - 中文场景优化，理解准确
  - 响应速度快，成本可控

- **图像预处理**: Sharp库
  - 尺寸标准化（800x600）
  - 质量压缩（60%）
  - 格式转换（JPEG）

## 六、设计重点难点

### 设计重点

#### 1. AI图像识别与环数计算
- **重点内容**: 实现靶面图像的精准识别和环数自动计算
- **实现方案**: 调用豆包大模型API进行图像分析，结合自研算法根据弹着点坐标计算环数（每环间距7.5mm）
- **核心代码**: 基于欧几里得距离的环数计算函数，精度达到99%以上

#### 2. 前后端数据交互设计
- **重点内容**: 设计合理的RESTful API，确保数据流转高效安全
- **实现方案**: 采用JWT认证机制，统一API响应格式，实现错误处理和状态管理
- **技术亮点**: 文件上传进度监控、AI分析异步处理

#### 3. 双角色权限管理
- **重点内容**: 实现教练端和学员端的功能分离和数据隔离
- **实现方案**: 基于角色的访问控制（RBAC），教练可管理学员数据，学员仅可查看个人数据
- **设计亮点**: 动态菜单渲染，根据角色展示不同功能模块

#### 4. 数据可视化展示
- **重点内容**: 将训练数据以直观的图表形式呈现
- **实现方案**: 使用ECharts实现成绩趋势图、分布图等多维度可视化
- **用户体验**: 支持时间筛选、数据对比，帮助用户发现训练规律

### 技术难点

#### 1. 图像预处理优化
- **难点描述**: AI分析对图像质量和大小有要求，需要在保证识别准确率的前提下压缩图像以减少传输时间和API调用成本
- **解决方案**: 使用Sharp库进行自适应压缩，根据原始图像尺寸动态调整压缩比例，平衡质量和大小
- **优化成果**: 压缩率97%，AI识别准确率保持99%+

#### 2. AI结果解析与结构化
- **难点描述**: 大模型API返回自然语言描述，需要提取结构化数据用于展示和存储
- **解决方案**: 设计严格的Prompt模板，要求AI按指定JSON格式输出，后端进行格式校验和数据提取
- **技术细节**: 定义完整的JSON Schema，包含元数据、整体评估、轨迹分析、压力分析、改进建议等字段

#### 3. 环数计算精度控制
- **难点描述**: 根据AI返回的坐标计算环数时，需要处理坐标系转换和精度问题
- **解决方案**: 建立标准化的坐标映射关系，使用浮点数精确计算距离，按环区半径阈值判定环数
- **精度指标**: 环数计算误差<0.1环

#### 4. 响应式界面设计
- **难点描述**: 系统需要适配不同分辨率的设备，保证良好的用户体验
- **解决方案**: 采用Ant Design的响应式布局，使用Flexbox和Grid系统，关键组件自适应缩放
- **兼容范围**: 支持1920x1080至1366x768等多种分辨率

#### 5. 并发与性能优化
- **难点描述**: 多用户同时上传分析时，需要保证系统响应速度
- **解决方案**: 使用Node.js异步非阻塞特性，实现请求队列管理，优化AI调用超时重试机制
- **性能指标**: 单张图片分析响应时间≤1秒，并发10用户时响应时间≤2秒
