# 软件应用与开发类技术设计文档

## 第一章 需求分析

### 1.1 开发背景与原因
射击训练数据管理面临四大核心痛点：

1. **数据管理混乱**：训练照片和分析结果分散存储，难以统一管理
2. **历史数据追踪难**：缺乏系统化的历史记录管理，无法进行长期对比分析
3. **多角色协作困难**：教练和学员数据隔离，无法进行有效的训练指导
4. **数据可视化缺失**：训练数据缺乏直观展示，难以发现规律和问题

### 1.2 目标用户分析
- **核心用户**：射击俱乐部教练、体校射击队教练、射击学员
- **次要用户**：射击爱好者、训练管理人员
- **用户规模**：预计单俱乐部50-200人，支持多俱乐部部署

### 1.3 竞品对标分析
| 对比维度 | 传统Excel管理 | 简单相册应用 | 本项目 |
| :--- | :--- | :--- | :--- |
| **核心功能** | 表格记录 | 图片存储 | 用户系统+AI分析+数据可视化 |
| **多角色支持** | 无 | 无 | 教练端/学员端分离 |
| **数据可视化** | 无 | 无 | 图表展示、趋势分析 |
| **AI辅助** | 无 | 无 | 智能分析、个性化建议 |
| **使用门槛** | 需培训 | 简单 | 界面友好，即装即用 |

**差异化定位**：专注于射击训练数据管理，提供多角色协作、AI辅助分析、数据可视化的轻量化解决方案。

### 1.4 核心功能与性能需求
| 核心痛点 | P0级功能 | 核心性能指标 |
| :--- | :--- | :--- |
| 数据管理混乱 | 用户系统+图片上传+AI分析 | 响应≤3秒，上传成功率≥99% |
| 历史数据追踪难 | 历史记录管理+筛选+导出 | 查询响应≤1秒，导出成功率≥99% |
| 多角色协作困难 | 教练端/学员端角色分离 | 角色切换流畅，权限控制准确 |
| 数据可视化缺失 | 图表展示+统计看板 | 图表渲染流畅，交互响应及时 |

### 1.5 技术可行性分析
- **前端技术**：React生态成熟，组件丰富，开发效率高
- **后端技术**：Node.js性能优异，适合I/O密集型应用
- **AI集成**：大模型API服务稳定，调用成本可控
- **部署方案**：支持本地部署和云服务，灵活适配

## 第二章 概要设计

### 2.1 项目总览
本项目面向射击训练**数据化、系统化、协作化**需求，构建基于Web的射击训练分析管理平台。采用前后端分离架构，前端使用React + TypeScript + Vite，后端使用Node.js + Express，集成AI大模型API实现智能分析。

### 2.2 系统架构设计

#### 2.2.1 三层架构
```
┌─────────────────────────────────────────────────────────────┐
│                        前端层 (React)                        │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│  │   教练端界面  │  │   学员端界面  │  │    数据可视化     │  │
│  └──────────────┘  └──────────────┘  └──────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                            ↕ RESTful API
┌─────────────────────────────────────────────────────────────┐
│                       后端层 (Node.js)                       │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│  │   用户认证    │  │   文件管理    │  │   AI分析调度     │  │
│  └──────────────┘  └──────────────┘  └──────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                            ↕
┌─────────────────────────────────────────────────────────────┐
│                       AI服务层                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
```

#### 2.2.2 AI模块集成架构
```
用户上传图片 → 后端接收 → 图像预处理 → AI分析API调用
                                               ↓
结果展示 ← 数据存储 ← 结果解析 ← 环数识别/姿势分析/建议生成
```

### 2.3 核心功能设计
1. **用户系统**：教练端/学员端注册登录，角色权限管理
2. **图片上传与AI分析**：支持靶面图片上传，集成AI分析API
3. **历史记录管理**：训练记录管理、筛选、搜索、导出
4. **数据可视化**：成绩趋势图表、统计看板、数据展示

### 3.3 AI核心算法流程

#### 3.3.1 靶面分析算法
```
输入: 靶面图片
  ↓
步骤1: 图像预处理
  - 尺寸标准化 (resize to 800x600)
  - 亮度对比度增强
  ↓
步骤2: 目标检测
  - 靶心定位 (center detection)
  - 弹孔识别 (bullet hole detection)
  ↓
步骤3: 环数计算
  - 距离测量 (distance from center)
  - 环数映射 (ring number mapping)
  ↓
步骤4: 姿势分析
  - 持枪姿势评估
  - 稳定性分析
  ↓
输出: {环数, 弹着点坐标, 姿势评分, 改进建议}
```

## 第五章 安装及使用

### 5.1 运行环境要求
- **硬件**：可联网电脑/服务器，4GB+内存
- **软件**：Node.js 16+，Chrome 90+/Edge 90+/Firefox 88+
- **网络**：稳定的互联网连接（用于AI分析）

### 5.2 部署步骤
```bash
# 1. 克隆项目代码
git clone <repository-url>

# 2. 后端部署
cd backend
npm install
# 配置环境变量 .env
node server.js

# 3. 前端部署
cd ..
npm install
npm run dev
```

### 5.3 使用流程
1. **教练注册** → 创建教练账号，设置密码
2. **学员注册** → 关联教练，完善信息
3. **上传训练** → 拖拽上传靶面图片，AI自动分析
4. **查看记录** → 历史数据管理，筛选导出
5. **统计分析** → 数据可视化，趋势分析

### 5.4 常见问题排查
| 问题 | 可能原因 | 排查步骤 |
| :--- | :--- | :--- |
| 无法登录 | 账号密码错误 | 检查输入，确认后端服务运行 |
| AI分析失败 | 网络/API密钥 | 检查网络连接和API密钥配置 |
| 图表不显示 | 浏览器兼容 | 刷新页面，检查浏览器版本 |
| 上传失败 | 文件过大 | 检查文件大小，压缩后重试 |
